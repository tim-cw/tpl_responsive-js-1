MIXIN TYPO

À partir de maintenant, nous avons modifié la méthode SCSS typo (dans styles/tools/typo.scss). La méthode s'attend à recevoir 4 paramètres , dont les deux derniers ne sont pas obligatoires :

  1. Grosseur du texte à la résolution la plus petite ( en px ).
  2. Grosseur du texte à la plus grande résolution ( en px ).
  3. Grandeur de la résolution la plus petite (en px, par défaut 480px ).
  4. Grandeur de la résolution la plus grande (en px, par défaut 1440px ).

Vous pouvez donc l'utiliser de deux manières différentes :

  • font-size: #{typo(18px, 32px)}
  • font-size: #{typo(18px, 32px, 480px, 1200px)}

Transition

La propriété transition doit avoir 3 valeurs pour fonctionner adéquatement : la vitesse de la transition (règle de pouce, jamais plus que 0.5s), les propriétés qui seront affectées par la transition (utilisez all pour affecter toutes les propriétés) et l'effet de transition (easing).

Pour conclure ce numéro, vous devez ajouter une transition sur le bouton qui affecte toutes les propriétés qui changent et qui dure 0.3 seconde avec un effet de transition ease-out .

Testez la propriété d'abord sur le survol du bouton. Observez ce qui se passe au survol, puis lorsque vous quittez le bouton. Déplacez ensuite la propriété dans la classe .button de base et observez les différences. Une fois la différence aperçue, dans le HTML, ajouter un paragraphe sous le bouton qui explique la différence entre l'ajout de la propriété transition directement sur le bouton ou en utilisant le survol du bouton.

BOUTON MENU MOBILE

Pour cette étape, vous pouvez réduire la largeur de votre écran et observer à quel moment l'espace devient trop restreint pour afficher à la fois la navigation et le logo sur une seule ligne. À ce stade, nous allons créer un media query qui masquera la navigation et fera apparaître un bouton de type hamburger.

Nous allons commencer par ajouter une icône de menu dans notre en-tête et nous assurer qu'elle est cachée par défaut.

Dans le HTML

  • Ajouter une balise <button> avec la classe menu-toggle qui contient l'icône de menu. C'est l'exemple parfait d'une situation où vous devriez utiliser une balise <button> plutôt qu'une balise <a>. Lorsqu'on clique dessus, nous ne sommes pas dirigés vers une autre page, il s'agit donc d'un <button>.

Dans le CSS (idéalement dans votre fichier scss qui gère les boutons)

  • Ajouter une règle pour .menu-toggle
  • Retirer la bordure et l'arrière-plan du bouton.
  • Régler la couleur pour --color-primary .
  • Régler la largeur de l'icône pour qu'elle soit de 32px . N'oubliez pas que notre icône a une largeur et une hauteur de 1em, ce qui signifie qu'elle prend la largeur et la hauteur du font-size de son parent.

Il est maintenant nécessaire de cacher le menu par défaut. La question à se poser est de savoir à quel endroit il faut ajouter la règle CSS pour cacher le menu. Doit-il être caché dans le fichier button ou dans le fichier header? Pour y répondre, il faut se demander si nous le cachons parce qu'il est situé dans l'en-tête, ce qui est effectivement le cas. Par conséquent, nous allons appliquer les styles pour le cacher dans le fichier header.scss .

À l'aide de l'imbrication, ciblez le menu-toggle lorsqu'il est dans le header et assurez-vous qu'il ne soit plus affiché en modifiant sa propriété display. 

        .header {
            . menu-toggle { 
                display: none; 
            } 
        }

Responsive

Nous sommes maintenant arrivés à l'étape où nous devons nous assurer que la navigation se cache et que le bouton de menu apparaît, lorsque l'espace disponible est insuffisant. En testant différentes résolutions, nous constatons que dans la version tablette (768px), l'espace devient insuffisant. 

            @media screen and (max-width: 768px) {
                
            }

Cette ligne permet de définir des règles ou des changements de propriété uniquement si votre écran est plus petit que 768px. Cependant, la valeur 768px sera utilisée à plusieurs reprises dans votre projet, il serait judicieux d'utiliser une variable. Nos variables 'breakpoints' sont conçues pour cette situation. 

            @media screen and (max-width: $breakpoint-sm) { 
                
            }

Utiliser cette ligne avec de l'imbrication dans votre menu-toggle pour modifier sa propriété display de manière à ce qu'il apparaisse. 

            .header { 
                [...] 
                .menu-toggle { 
                    display: none; 
                    @media screen and (max-width: $breakpoint-sm) {
                         display: block; 
                    } 
                } 
            }

Utiliser la même technique pour que la navigation ait l'effet contraire et se cache lorsque l'écran est plus petit que 768px.

Nos cartes sont actuellement assez grandes, il serait maintenant judicieux de préparer nos colonnes. Pour ce faire, ajouter un nouveau div qui englobe vos trois cartes et qui a la classe .cards .

RESPONSIVE

Nous allons terminer cet exercice en ajoutant les règles de media query pour garantir que lorsque vous changez de résolution, vos colonnes passent de 3 colonnes en desktop à 2 sur tablette, puis finalement à 1 sur mobile.

Dans votre règle de .cards, imbriquez un media query qui modifie les styles pour 768px (n'oubliez pas d'utiliser vos variables).

  • Réglez le grid-template-columns pour avoir une seule colonne d'une fraction .

En testant, vous devriez constater que vous avez une seule colonne lorsque vous êtes dans une taille d'écran inférieure à 768px.

Sous votre media query de 768px, ajoutez un nouveau media query qui modifie les styles à 1024px.

  • Réglez le grid-template-columns pour avoir deux colonnes d'une fraction .

Très bien, cela fonctionne. Oh, en fait, si vous regardez en dessous de 1024px, vous aurez maintenant toujours 2 colonnes. C'est le poids des sélecteurs qui crée ce problème. Dans l'approche "desktop first" , l'ordre de nos media queries devrait toujours être du plus grand au plus petit . Inversez l'ordre de vos media queries.
